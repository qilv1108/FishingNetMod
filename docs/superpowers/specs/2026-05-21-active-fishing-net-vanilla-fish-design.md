# 主动捕鱼接入原版鱼池设计

**目标：** 主动撒网捕获的鱼类与星露谷原版钓鱼结果一致，让河里、海里、湖里等水域按游戏自身的位置、季节、天气和时间规则产出鱼。

## 范围

本次只替换主动撒网 MVP 的固定捕获池，不实现被动放置、配方、任务链、贴图或 Content Patcher 内容包。

## 当前问题

`FishingNetMod/Mechanics/ActiveFishingNet.cs` 现在使用固定 `catchPool`：

- 不区分河流、海洋、湖泊等地点。
- 不跟随季节、天气、时间。
- 会出现“海里撒网也可能出河鱼”的问题。

## 方案

采用方案 1：主动撒网直接接入原版 `GameLocation.getFish(...)` 逻辑。

新增 `VanillaFishProvider`，由它负责从当前 `GameLocation` 和玩家面向的水格获取原版鱼类结果。`ActiveFishingNet` 不再维护固定鱼池，只负责：

1. 判断玩家是否手持捕鱼网。
2. 判断面前 tile 是否为水域。
3. 根据捕鱼网等级计算捕获次数。
4. 每次调用 `VanillaFishProvider` 获取一条原版鱼。
5. 把鱼放入背包，背包满则掉落到玩家脚边。
6. 扣除体力并显示 HUD 提示。

## 行为规则

- 铜网捕获 2-3 条。
- 铁网捕获 3-4 条。
- 金网捕获 4-5 条。
- 铱星网捕获 5-7 条。
- 捕鱼网等级不改变鱼池本身，只改变捕获数量和体力消耗。
- 河里撒网应接近河钓结果。
- 海里撒网应接近海钓结果。
- 湖里、矿洞、姜岛等其他可钓鱼水域也应使用原版地点鱼池。
- 如果原版调用没有返回有效鱼类，本次捕获跳过；若整次撒网都没有鱼，显示“没有捕到鱼。”。

## 文件变更

- 修改 `FishingNetMod/Mechanics/ActiveFishingNet.cs`
  - 删除固定 `catchPool`。
  - 通过 `VanillaFishProvider` 获取鱼类。
- 新增 `FishingNetMod/Mechanics/VanillaFishProvider.cs`
  - 封装原版 `GameLocation.getFish(...)` 调用。
  - 隔离 SMAPI/游戏 API 参数细节，方便运行时调试。
- 测试侧优先覆盖可测试的控制流：无有效鱼时返回空结果、等级捕获次数仍保持原数据。

## 验证方式

运行时验证必须进入游戏存档：

1. 在 SMAPI 控制台执行 `fishing_net give copper`。
2. 手持铜网在河流旁撒网，观察捕获鱼类应符合当前季节/天气/时间的河流鱼。
3. 手持铜网在海边撒网，观察捕获鱼类应符合当前季节/天气/时间的海鱼。
4. 在非水域使用仍提示“这里不能撒网。”。
5. 捕获数量仍为铜网 2-3 条，并扣除 10 体力。

## 非目标

- 不强制产生垃圾。
- 不进入钓鱼小游戏。
- 不实现宝箱、钓竿、鱼饵、浮标完整效果。
- 不新增自定义鱼池。
