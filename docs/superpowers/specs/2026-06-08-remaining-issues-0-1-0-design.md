# Fishing Net Mod 0.1.0 剩余问题处理设计

日期：2026-06-08

## 目标

把当前项目推进到 0.1.0 测试版发布状态。0.1.0 的目标不是完整正式版，而是让核心玩法能被稳定安装、启动、手动验证和分发。

## 范围

0.1.0 只处理发布闸门问题：

- 完成游戏内手动验证。
- 清理或归档未跟踪文件。
- 明确多人、语言、动画、固定垃圾概率等已知限制。
- 确保根目录 `dotnet test` 通过。
- 确保生成的 `FishingNetMod 0.1.0.zip` 可安装测试。

以下内容不进入 0.1.0 代码范围：

- 多人模式完整权限与归属规则。
- 全量 i18n 和 C# 文本迁移。
- 主动撒网动画增强。
- 被动渔网固定 10% 垃圾概率。
- 宝箱式收网界面。

## 发布闸门设计

### 1. 游戏内手动验证

使用 `docs/manual-test-checklist.md` 作为 0.1.0 发布前清单。测试必须覆盖加载与命令、主动撒网、被动放置、任务与解锁、打包安装五类核心路径。

0.1.0 的通过标准：

- SMAPI 加载 Mod 无红色报错。
- `fishing_net give copper|iron|gold|iridium` 均可获得对应渔网。
- 第二级渔网显示为 `Silver Fishing Net`。
- 主动撒网在河流、湖泊、海洋至少各验证一次。
- 非水域主动撒网不消耗体力。
- 被动渔网可放置、跨天产出、挑战收网并返还渔网。
- 被动渔网地图贴图使用自定义渔网贴图。
- 主要配方解锁路径至少通过调试或真实流程验证。
- 发布 zip 安装到 Mods 目录后可正常加载。

### 2. 未跟踪文件处理

当前未跟踪文件需要在发布前处理：

- `FishingNetMod-完成报告.md`
- `task-mpwcojzx4hw9o.jpg`

处理规则：

- 如果 `FishingNetMod-完成报告.md` 是需要保留的项目记录，移动到 `docs/` 并命名为发布或完成报告；否则不纳入发布。
- 如果 `task-mpwcojzx4hw9o.jpg` 是正式素材，移动到 `FishingNetMod/[CP] FishingNetMod/assets/` 并更新引用；如果只是参考或临时图，不纳入发布。
- 发布前 `git status --short` 不应包含无法解释的临时文件。

### 3. 多人模式策略

0.1.0 按单人测试版发布。多人模式标记为实验性，不承诺完整行为。

当前代码保留：

- 每个玩家只能放置一个网的基础限制。
- 被动网数据记录 `OwnerId`。

0.1.0 不改动：

- 其他玩家是否能收网。
- 被动产出任务进度归属。
- 多人同步与权限提示。

发布说明必须写明：0.1.0 主要按单人流程验证，多人模式需要后续版本专门设计和测试。

### 4. 文本与 i18n 策略

0.1.0 不做全量文本统一。当前允许存在：

- Content Patcher 物品、描述、邮件使用英文。
- C# HUD 和错误提示使用中文。

发布说明必须写明：语言文本尚未统一，后续版本会迁移到 SMAPI translation 或补充中文 Content Patcher i18n。

### 5. 主动撒网表现策略

0.1.0 保留当前主动撒网表现：

- 水域检测。
- 体力消耗。
- 水花音效。
- 数字挑战菜单。
- HUD 结果提示。

不新增动画系统。动画和视觉反馈增强作为后续体验优化，不阻塞测试版发布。

### 6. 被动网垃圾概率策略

0.1.0 保持当前被动产出实现：复用原版 `getFish` 行为，不单独保证固定 10% 垃圾概率。

发布说明必须写明：被动渔网产出暂按原版钓鱼结果生成，固定垃圾概率是后续可选设计项。

### 7. 自动化验证

发布前必须从仓库根目录运行：

```powershell
dotnet test
```

通过标准：

- 测试全部通过。
- 当前基线为 74 个测试。
- 构建过程中能生成 `FishingNetMod/bin/Debug/net6.0/FishingNetMod 0.1.0.zip`。

### 8. 发布说明

0.1.0 发布说明至少包含：

- 主要功能：四级渔网、主动撒网、被动放置、数字挑战、任务解锁、Content Patcher 物品与配方。
- 安装方式：安装 SMAPI、Content Patcher，把 zip 解压到 Mods 目录。
- 测试状态：自动测试结果和手动测试清单状态。
- 已知限制：多人实验性、文本未统一、主动动画较基础、被动垃圾概率未固定、收网流程为挑战直接发放而非宝箱界面。

## 验收标准

0.1.0 可以发布测试版，当且仅当：

- `dotnet test` 通过。
- `FishingNetMod 0.1.0.zip` 存在并能安装。
- 手动测试清单的核心路径已完成并记录结果。
- 未跟踪临时文件已处理。
- 发布说明包含已知限制。

## 后续版本候选任务

0.1.1 或后续版本可按优先级拆分：

- 多人权限与任务归属。
- 统一 i18n。
- 主动撒网动画与水面反馈。
- 固定垃圾概率及对应测试。
- 宝箱式收网界面。
- 发布文档与设计文档一致性整理。
