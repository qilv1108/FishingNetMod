## 一、项目总体情况

这是一个《星露谷物语》SMAPI Mod 项目，目标是实现“捕鱼网”功能。

当前项目结构大致如下：

- `FishingNetMod/`
  - C# Mod 主项目
  - `ModEntry.cs` 作为 SMAPI 入口
  - `Data/` 捕鱼网等级数据
  - `Items/` 捕鱼网物品工厂与 ID
  - `Mechanics/` 主动撒网、被动渔网、鱼类获取、渲染、挑战逻辑
  - `Menus/` 捕鱼/收网挑战菜单
  - `Quests/` 任务进度与配方解锁逻辑
  - `[CP] FishingNetMod/` Content Patcher 内容包
- `FishingNetMod.Tests/`
  - xUnit 测试项目
  - 覆盖结构、物品、机制、任务、菜单等模块
- `docs/superpowers/`
  - 设计文档与实施计划
- `2026-05-20-fishing-net-implementation-design.md`
  - 原始设计文档
- `SMAPI/`、`.gstack-install/`、`.bun-ascii/`
  - 外部参考或本地工具目录，已在 `.gitignore` 中忽略
- `task-mpwcojzx4hw9o.jpg`
  - 当前未跟踪文件，可能是任务相关图片或临时文件

### 4. 收网流程与原设计不完全一致

原设计文档中写的是：

- 收网时弹出类似宝箱界面
- 显示里面的鱼
- 玩家点击取走所有鱼
- 网回到背包

当前实现是：

- 面向渔网按交互键
- 弹出数字挑战菜单
- 挑战成功后直接把所有鱼和网放入背包或掉落

这不是错误，但与原设计有差异。

需要决定：

- 保持当前“挑战收网”的玩法
- 还是改回“宝箱式收获界面”
- 或二者结合：挑战成功后打开收获界面

### 10. 被动产出没有单独实现“10% 垃圾概率”

设计文档提到：

- 被动放置有 10% 概率出垃圾代替鱼

当前被动产出调用的是：

```csharp
fishProvider.GetFish(...)
```

也就是复用原版 `getFish`。

这可能已经包含原版垃圾逻辑，但不是设计文档里明确的“被动网固定 10% 垃圾概率”。

建议确认设计：

- 如果接受完全复用原版钓鱼逻辑，则当前可以保留
- 如果要严格实现 10% 垃圾概率，需要额外添加逻辑和测试

- 钓鱼 1 级后排队铜网邮件

测试中也明确验证钓鱼等级要求为 1。

这可能是后续设计调整，但建议在最终文档中同步，避免“代码是 1 级、文档是 2 级”的不一致。
